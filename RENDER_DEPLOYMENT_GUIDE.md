# 🚀 Render.com Deployment Guide for SyncScript

## Prerequisites
- GitHub account with `syncscriptE` repository
- Render.com account (free tier available)
- Environment variables ready

## Step 1: Deploy to Render.com

### 1.1 Go to Render.com
1. Visit [render.com](https://render.com)
2. **Sign in** with your GitHub account

### 1.2 Create New Blueprint
1. Click **"New +"** → **"Blueprint"**
2. **Connect your repository**: `syncscriptE`
3. Render will automatically detect your `render.yaml` file
4. Click **"Apply"**

### 1.3 Set Environment Variables
In the Render.com dashboard, set these environment variables:

#### Required Variables:
```
NODE_ENV=production
JWT_SECRET=your-super-secret-jwt-key-minimum-32-characters
OPENAI_API_KEY=sk-proj-your-openai-api-key-here
EMAIL_USER=your-gmail-address@gmail.com
EMAIL_APP_PASSWORD=your-gmail-app-password
GOOGLE_CLIENT_ID=your-google-client-id.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=your-google-client-secret
GOOGLE_REDIRECT_URI=https://stringerc.github.io/syncscriptE/google-calendar
PLAID_CLIENT_ID=68af6e15d275380025927fe9
PLAID_SECRET=your-plaid-secret
PLAID_ENVIRONMENT=sandbox
FRONTEND_URL=https://stringerc.github.io/syncscriptE
```

#### Database Variables (Auto-generated by Render):
```
DATABASE_URL=postgresql://username:password@host:port/database
```

## Step 2: Wait for Deployment
- **Deployment time**: 5-10 minutes
- **Check logs** in Render.com dashboard
- **Test health endpoint**: `https://syncscript-backend.onrender.com/health`

## Step 3: Update GitHub Pages
Once Render.com is deployed, the GitHub Pages will automatically use the Render.com backend.

## Step 4: Test Your App
1. **Visit**: `https://stringerc.github.io/syncscriptE/`
2. **Register** a new account
3. **Login** and test functionality

## Troubleshooting

### Common Issues:
1. **Build fails**: Check environment variables are set
2. **Database connection fails**: Wait for PostgreSQL to be ready
3. **API calls fail**: Check CORS settings and frontend URL

### Support:
- Render.com documentation: https://render.com/docs
- Check deployment logs in Render.com dashboard

## Success Indicators:
- ✅ Backend health check returns 200
- ✅ Registration/login works on GitHub Pages
- ✅ All features functional
